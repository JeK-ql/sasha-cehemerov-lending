# Лендинг Sasha Chemerov — специфікація дизайну

**Дата:** 2026-05-22
**Проєкт:** Односторінковий лендинг для продажу одного дропу — оверсайз-футболки
**Артист:** Саша Чемеров · гурт «Димна Суміш»
**Товар:** футболка «too much яром too much долиною», оверсайз, один розмір, **2200 ₴**
**Домен:** https://isusneisus.com/
**Стек:** Next.js (App Router) · Vercel · SSG

---

## 1. Концепція

Ультрамінімалістичний односторінковий лендинг (SPA-відчуття) із виїзною модалкою для
моментальної покупки. Сторінка спроєктована **виключно для фанатів**, які переходять за прямим
посиланням або сканують QR-код на концертах, у мерч-зонах і соцмережах. Випадкових відвідувачів
немає, тому сайт **не «переконує» купити** — він працює як ексклюзивна цифрова галерея одного
арт-обʼєкта.

- **90% трафіку — смартфон.** Дизайн mobile-first, десктоп — вторинний.
- Уся логіка покупки схована в модалці — без переходів на нові сторінки.
- Мова інтерфейсу — українська.

## 2. Візуальний напрямок — Swiss Neo-Brutalism

Сухий, безкомпромісний, ультра-стильний дизайн у дусі швейцарської школи плаката. Гігантський
контраст масивного чорного тексту й чистого білого простору; фотографії — головні арт-обʼєкти.
Жодних тіней, градієнтів-прикрас, заокруглень-прикрас. Лише контраст, сітка й одна гаряча
акцентна пляма. Колізія брутального рок-вайбу та швейцарського мінімалізму — це і є ідея.

Референси: Highsnobiety, Grailed, Balenciaga.

## 5. Структура головної сторінки

### 5.1. Хедер (закріплений)

Тонкий рядок угорі, не скролиться.

- **Зліва:** логотип `logo.png` як зображення, перефарбоване в монохром через CSS `filter` (`brightness(0)` — чорний на світлому хедері). Фінальний логотип — за клієнтом.
- **Справа:** технічний маркер `DROP 01 // ONE SIZE (OVERSIZE)`, колір `#7A7A7A`.
- Мобайл: компактно (~12px), межа знизу 1.5px.
- Десктоп: написи рознесені по краях, багато повітря.

### 5.2. Герой — «Афіша» (напрямок A)

- Фото Чемерова на весь кадр (full-bleed).
- Поверх — гігантський заголовок «TOO MUCH ЯРОМ TOO MUCH ДОЛИНОЮ», зрізаний краєм; слово
  «ДОЛИНОЮ» — червоне. Енергія гіг-постера.
- Затемнення-градієнт знизу для читабельності білого тексту.
- Вордмарк зверху, скрол-підказка «↓ Артефакт».
- Десктоп: фото 16:9, заголовок знизу-зліва. Мобайл: вертикальний кроп по центру фігури.

### 5.3. Галерея «Артефакт»

Показ товару.

- **Мобайл:** авто-слайдер — сам міняється кожні ~2,8 с, плавний кросфейд + лёгкий зум.
  Без свайпу. 4 кадри: товар-перед → товар-спина → Чемеров-перед → Чемеров-спина.
  Прогрес-смужка (червона) + підпис кадру.
- **Десктоп:** диптих — 2 фото товару (front + back) поруч. При наведенні курсору кадр
  кросфейдить на Чемерова в цій футболці (front→Чемеров-перед, back→Чемеров-спина).
- Фото товару — чорна футболка на суцільному чорному тлі → ефект «чорний квадрат-постер».

### 5.4. Інфо + CTA

- Рядок специфікації (графіт): `HEAVYWEIGHT COTTON · 100% БАВОВНА · OVERSIZE · ОДИН РОЗМІР`.
- Кнопка **«ЗАМОВИТИ — 2200 ₴»**, колір `#F14933`, білий текст.
  - Мобайл: закріплена внизу екрана (sticky bottom bar), під великим пальцем.
  - Десктоп: широка плашка/банер; при наведенні плавно інвертується в чорну.

### 5.5. Футер

Мінімальний: `ДИМНА СУМІШ © 2026` · `НОВА ПОШТА` · `WAYFORPAY`.
Посилання на окремі юридичні сторінки: **Публічна оферта** (`/offer`) ·
**Умови повернення** (`/returns`).

## 6. Чекаут-модалка

Кнопка «Замовити» не веде на нову сторінку — миттєво відкривається модалка.

**Інверсія:** тло модалки — глибокий матовий чорний `#0D0D0D`, текст і лінії — білі та графітові.
Потужний контраст із білим сайтом.

### Поведінка

- **Мобайл:** шторка виїжджає знизу вгору (як картка Apple Wallet / системна шторка iOS),
  закриває ~95% екрана; зверху видно затемнений сайт. Грабер, ✕ у правому верхньому куті.
- **Десктоп:** вертикальна панель ~⅓ ширини виїжджає з правого краю; решта сайту затемнюється
  напівпрозорою чорною пеленою.

### Вміст форми

Поля без рамок — лише тонкі горизонтальні лінії; активна лінія підсвічується червоним.

1. **Зведення замовлення:** мініатюра товару + назва + `2200 ₴`.
2. **Блок 01 — Хто ти:**
   - Прізвище та імʼя
   - Телефон (тап → цифрова клавіатура на мобайлі)
   - **E-mail** — підпис «сюди прийде чек і підтвердження замовлення». На цю адресу
     WayForPay автоматично надсилає фіскальний чек (див. 8.1).
3. **Блок 02 — Куди везти:** інтеграція **Нової Пошти**
   - Місто — автопідказка при введенні
   - Відділення / поштомат — вибір зі списку
   - Без індексів і домашніх адрес
4. **Кнопка оплати:** біла, чорний текст «ОПЛАТИТИ КАРТОЮ», підпис `WAYFORPAY · APPLE PAY · GOOGLE PAY`.
   Відкриває захищений віджет **WayForPay** просто в модалці (оплата в один клік).

## 8. Технічна архітектура та інтеграції

**Стек:** Next.js (App Router) на Vercel. Головна та юридичні сторінки — **SSG**
(статична генерація): швидко, дешево, добре індексується. Динаміка — у Route Handlers
(серверні функції Vercel). Усі секрети — у змінних оточення Vercel, ніколи на клієнті.

**Юридичні сторінки:** «Публічна оферта» (`/offer`) та «Умови повернення» (`/returns`) —
окремі статичні маршрути, посилання у футері. Контент надає клієнт/юрист — потрібен і для
підключення WayForPay.

### 8.1. WayForPay — оплата та чек

- Покупка — через **віджет WayForPay** просто в модалці, без переходу на сторонні сторінки.
- Серверний Route Handler формує `merchantSignature` (HMAC-MD5); секретний ключ — лише на сервері.
- У запит на оплату передаємо назву товару, кількість, ціну та **e-mail покупця**.
- `serviceUrl` — серверний колбек: перевіряє підпис, фіксує успішну оплату, тригерить Telegram.
- `returnUrl` — повернення на сайт після оплати.

**Чек — окремо робити не треба.** WayForPay має вбудований **програмний РРО (пРРО)**:
автоматично фіскалізує оплату, **надсилає фіскальний чек покупцю на e-mail** і сам подає
Z-звіти в податкову. Безкоштовно. Для покупця більше нічого не потрібно.
Умови: фіскалізацію треба **увімкнути в кабінеті WayForPay**; продавець має бути
зареєстрованим ФОП/ТОВ. За API (`apiVersion=2`) у відповіді доступні `prroLink` і
`prroNumber` — посилання й номер чека.

### 8.2. Нова Пошта — доставка

- API Нової Пошти: автопідказка міста при введенні + список відділень/поштоматів.
- Запити йдуть через серверний проксі (Route Handler), щоб не світити API-ключ на клієнті.

### 8.3. Telegram — сповіщення менеджерам

- Після успішної оплати (колбек WayForPay) серверний обробник надсилає повідомлення в
  **Telegram-чат менеджерів** через бота.
- Вміст: товар, ім'я, телефон, e-mail, місто, відділення Нової Пошти, сума, час.
- Потрібно: токен Telegram-бота + ID чату менеджерів (у змінних оточення Vercel).

### 8.4. Зберігання замовлень

Окремої БД немає — джерело правди про замовлення = Telegram-чат менеджерів + кабінет
WayForPay (історія платежів). За потреби пізніше можна додати таблицю/БД.

## 10. Адаптив

Mobile-first (90% трафіку). Десктоп — розширення тієї самої композиції: герой стає
ширшим, галерея — диптихом, модалка — бічною панеллю. Жодних окремих макетів — одна система.

---

## 11. Відкриті питання до етапу збірки

Вирішено: стек (Next.js/Vercel/SSG), домен (`isusneisus.com`), куди йдуть замовлення
(Telegram + кабінет WayForPay), чек (пРРО WayForPay).

Лишається закрити:

- Кастомний леттеринг заголовка дропу — хто і коли робить.
- Фінальний логотип із «димна суміш» — за клієнтом.
- Контент юридичних сторінок (оферта, умови повернення) — за клієнтом/юристом;
  потрібен і для підключення WayForPay.
- Доступи й секрети: merchant-акаунт WayForPay + увімкнена фіскалізація;
  токен Telegram-бота + ID чату; ключ API Нової Пошти.
- Перевірити, що згенеровані фото товару не спотворили дрібний текст принту.
