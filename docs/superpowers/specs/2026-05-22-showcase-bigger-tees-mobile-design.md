# Showcase: великі футболки + більший заголовок (мобільний фокус)

**Дата:** 2026-05-22
**Статус:** затверджено до планування
**Область:** компонент `Showcase` лендингу. Інші секції не зачіпаються.

## Проблема

На телефоні поточний Showcase виглядає слабко:

- Дві футболки стоять у ряд 50/50, тож кожна виходить дрібною.
- Головний заголовок («too much яром / too much долиною») замалий, а підпис «too much» губиться біля великого слова.

Користувач хоче помітний, «постерний» екран із фокусом на мобільному форматі.

## Рішення (затверджено)

### 1. Футболки — діагональний колаж із перекриттям

- Контейнер `.tees` перестає бути flex-рядком 50/50 і стає **позиційною сценою** (`position: relative`).
- Лишаються ті самі **дві** футболки: перед (`/front-back-without-bg.png`) і спина (`/back-without-bg.png`). Третього зображення немає.
- Кожна футболка займає **≈75% ширини сцени** (¾ ширини екрана на телефоні).
- Розкладка по діагоналі з перекриттям:
  - **Перед** (терновий вінець) — вгорі-зліва, `z-index: 2`, спереду стоса.
  - **Спина** — внизу-справа, `z-index: 1`, визирає з-під переду.
- Вертикальний зсув між футболками підбирається так, щоб **обидва принти лишались повністю видимими** (принт спини не має ховатись під переднею футболкою).
- Числові підписи «01 — Перед» / «02 — Спина» **прибираються** — на щільному колажі вони лише засмічують. Лишається один рядок специфікації `OVERSIZE · 100% Бавовна · Heavyweight`.

### 2. Заголовок — компонування «too much окремим рядком»

- Структура та сама: дві пари рядків — `TOO MUCH` / `ЯРОМ` та `TOO MUCH` / `ДОЛИНОЮ` (друге слово червоне, `--red`).
- `TOO MUCH` стає **окремим читабельним рядком над кожним словом** (а не дрібним інлайн-елементом біля нього):
  - шрифт IBM Plex Mono, верхній регістр, розрядка літер;
  - колір `--ink` (не приглушений сірий) — щоб напис не губився;
  - розмір відчутно більший за нинішній (нинішнє `clamp(9px, 1.8vw, 13px)` замале).
- Великі слова (`.display`, Montserrat 900) помітно більші. Нижню межу `clamp` на телефоні підняти з `24px` до приблизно `34–38px`, vw-складову теж угору. Точні значення підбираються візуально під час реалізації.
- Заголовок лишається по центру, над колажем футболок.

### 3. Мобільний формат / один екран

- Лендинг лишається **односкрінним**: `.shell` — flex-колонка, CTA закріплена знизу на телефоні.
- Сцена футболок масштабується за `min(ширина-залежний розмір ~75vw, висота-залежний розмір)`, щоб великі футболки **не обрізались** за хедером і закріпленою кнопкою CTA.
- Якщо екран зовсім низький і контент усе одно не вміщується — дозволити **м'який вертикальний скрол** замість обрізання. Зараз `Showcase` має `overflow: hidden`, який ріже контент; цю поведінку треба пом'якшити.
- Десктоп: та сама розкладка, просто більший масштаб; перевірити, що нічого не ламається.

## Файли, які змінюємо

- `components/Showcase/Showcase.module.css` — основні зміни: `.tees` → позиційна сцена з перекриттям; `.tee` → ~75% ширини, абсолютне позиціювання, `z-index`; розміри заголовка `.small` / `.big`; видалення стилів підписів `.cap`.
- `components/Showcase/Showcase.tsx` — прибрати `<figcaption>` з обох `<figure>`; оновити атрибут `sizes` у `Image` (тепер кожна футболка ≈75vw); за потреби — дрібна правка розмітки під позиційну сцену.
- За потреби — `app/page.module.css` / `.showcase` overflow, якщо м'який скрол потребує правки на рівні shell.

## Поза областю змін

- Header, OrderBar (CTA), Footer, чекаут-флоу — не чіпаємо.
- Шрифти: блеклетер `Germanica` **не повертаємо**; працюємо наявними Montserrat / IBM Plex Mono / Oswald / Inter.
- Самі зображення футболок не перемальовуємо — використовуємо наявні PNG без фону.

## Критерії готовності

- На телефоні (≈375–414px ширини) кожна футболка займає ≈¾ ширини екрана, вони перекриваються по діагоналі, перед — спереду й вище.
- Обидва принти (вінець спереду, текст ззаду) повністю видно.
- Заголовок помітно більший за поточний; «TOO MUCH» читається окремим рядком і не губиться.
- Контент не обрізається за хедером чи закріпленою кнопкою CTA; за потреби сторінка м'яко скролиться, а не ріже футболки.
- Десктопна версія лишається цілою.
- `npm run build` проходить без помилок.
